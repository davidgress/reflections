Reflection - section 8
How could having easy access to the entire history of a file make you a more 
efficient programmer in the long term?

Learn from mistakes. Review changes to see how issue evolved over time. Reestablish an old piece
of code. Find out the cause of a current bug by isolating new code that was added.


Reflection - section 15
What do you think are the pros and cons of manually choosing when to create a commit, 
like you do in Git, vs having versions automatically saved, like Google Docs does?
Pros - more control. keep bugs out.
Cons - might forget, too many versions get saved.

Reflect: section 18
Why do you think some version control systems, like Git, allow saving multiple files in one commit, 
while others, like Google Docs, treat each file separately?
To better meld the changes together since a change in one file could affect the other file.

Reflect: section 22
How can you use the commands git log and git diff to view the history of files?
log tells you all the changes in the repository. diff can be used to get the changes between two versions.
